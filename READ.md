# Задача: начать разработку мобильного приложения «Лесной гримуар»

## Общая идея

Нужно создать мобильное приложение в формате личного дневника / гримуара лесной ведьмы. Это не эзотерический калькулятор и не приложение для гаданий, а атмосферный личный дневник с красивым оформлением, сезонными ритуалами, заметками, желаниями, рецептами чаёв, Колесом года и ежедневными картами-настроениями.

Главный акцент приложения — оформление, атмосфера и ощущение личной магической книги.

Приложение должно быть тёмным, уютным, лесным, с преобладанием глубоких зелёных оттенков, рисованной стилистики, папоротников, мха, свечей, луны, старой бумаги, лесных тропинок, фонарей и природных деталей. Избегать яркой мультяшности, стерильного Material UI и дешёвой «эзотерики».

## Целевая аудитория

Женщины и девушки, которым близка эстетика леса, ведьмовства, уютных ритуалов, дневников, травников, сезонности, чая, свечей, луны и личных записей.

Приложение должно ощущаться не как утилита, а как вещь, которую хочется открыть вечером с чашкой чая.

## Технологическая задача

Создать стартовую структуру приложения.

Желательно использовать:

* мобильный-first подход;
* Capacitor / PWA / Android-friendly архитектуру;
* локальное хранение данных на первом этапе;
* простую расширяемую структуру разделов;
* аккуратную компонентную архитектуру.

Если стек уже выбран — придерживаться существующего. Если нет, предложить оптимальный стек для быстрого MVP.

## Основные разделы приложения

### 1. Главный экран

Главный экран должен быть атмосферной «обложкой» или входом в гримуар.

На нём:

* приветствие;
* текущая дата;
* текущий сезон;
* ближайший праздник Колеса года;
* карта дня;
* быстрые кнопки к основным разделам.

Визуально: тёмный зелёный фон, рисованные ветви / папоротники / луна / мягкое свечение.

### 2. Колесо года

Раздел с 8 сезонными праздниками:

* Самайн
* Йоль
* Имболк
* Остара
* Белтейн
* Лита
* Лугнасад
* Мабон

Для каждого праздника нужна карточка с:

* названием;
* датой;
* кратким описанием;
* символами;
* цветами;
* растениями;
* идеями для личного ритуала;
* заметками пользователя;
* возможностью создать запись за конкретный год.

Важно: это не религиозный справочник, а уютный сезонный дневник.

### 3. Напоминания

Добавить базовую систему напоминаний:

* за 7 дней до праздника;
* за 3 дня;
* в день праздника;
* пользовательские напоминания.

Тексты уведомлений должны быть атмосферными, например:

«Лес шепчет: до Йоля осталось 7 ночей.»

«Сегодня Самайн. Хороший день, чтобы отпустить старое.»

На MVP можно подготовить структуру данных и UI, даже если push/local notifications будут подключены позже.

### 4. Карта дня

Каждый день пользователь получает одну карту.

Это не гадание, а мягкий совет / настроение / маленькое задание.

Пример карты:

Название: Папоротник
Текст: «Не всё должно быть видно сразу. Иногда рост происходит в тени.»
Тип: совет
Категория: растения

Категории карт:

* растения;
* животные;
* луна и небо;
* предметы;
* времена года;
* праздники Колеса года;
* погодные состояния.

Нужно заложить структуру для 60–100 карт, но для MVP достаточно 10–15 тестовых.

Функции:

* получить карту дня;
* сохранить историю карт;
* открыть архив прошлых карт;
* отметить карту как понравившуюся.

### 5. Дневник

Раздел личных записей.

Поля записи:

* заголовок;
* текст;
* дата;
* настроение;
* теги;
* фото;
* связь с праздником Колеса года, если есть.

Настроения:

* туман;
* дождь;
* свеча;
* лес;
* гроза;
* звёзды;
* тишина;
* костёр.

### 6. Книга желаний

Раздел желаний / хочух.

Концепция: каждое желание — это «семя».

Поля:

* название желания;
* описание;
* дата создания;
* статус: посажено / растёт / исполнилось / отпущено;
* фото;
* заметки;
* дата исполнения.

Визуально можно показывать желания как маленький сад.

### 7. Травник и рецепты чаёв

Раздел личных рецептов.

Поля:

* название рецепта;
* ингредиенты;
* описание;
* настроение;
* сезон;
* фото;
* заметки.

Примеры категорий:

* чай для дождя;
* чай для сна;
* зимний чай;
* чай для вдохновения;
* согревающий напиток;
* выпечка;
* варенье.

Важно: не давать медицинских обещаний. Это личные уютные рецепты, а не лечебник.

### 8. Сокровища леса

Фотозаметки / коллекция вдохновения.

Пользователь может сохранять:

* листья;
* цветы;
* камни;
* лесные тропинки;
* облака;
* книги;
* свечи;
* уютные моменты.

Поля:

* фото;
* подпись;
* дата;
* место, опционально;
* теги.

## Визуальный стиль

Общее направление:

* тёмный лесной интерфейс;
* глубокий зелёный как основной цвет;
* приглушённое золото / латунь для акцентов;
* рисованные элементы;
* тонкие рамки;
* старые страницы;
* папоротники;
* мох;
* луна;
* свечи;
* фонари;
* травник;
* лесная ведьма;
* Колесо года.

Избегать:

* ярких кислотных цветов;
* белого стерильного интерфейса;
* типичной магии с черепами, пентаграммами и агрессивной мистикой;
* перегруженных эффектов;
* дешёвого fantasy UI.

Интерфейс должен быть мягким, тёмным, женственным, лесным и немного сказочным.

## Навигация

Предложить структуру навигации:

* Главная
* Дневник
* Колесо года
* Карта дня
* Ещё

В разделе «Ещё»:

* желания;
* травник;
* сокровища;
* настройки;
* архив.

## Что нужно сделать на первом этапе

1. Создать архитектуру проекта.
2. Подготовить основные экраны.
3. Создать дизайн-систему:

   * цвета;
   * шрифтовые стили;
   * карточки;
   * кнопки;
   * рамки;
   * фоновые декоративные элементы.
4. Создать mock data для:

   * праздников Колеса года;
   * карт дня;
   * рецептов;
   * дневниковых записей.
5. Реализовать базовую навигацию.
6. Реализовать главный экран.
7. Реализовать экран Колеса года.
8. Реализовать экран Карты дня.
9. Подготовить структуру локального хранения.
10. Описать дальнейший план развития.

На выходе нужен рабочий стартовый прототип приложения с красивой атмосферной основой, а не просто набор пустых экранов.

Главное ощущение:

«Я открываю личную лесную книгу, где можно хранить мысли, желания, сезонные заметки, рецепты и маленькие ритуалы.»
