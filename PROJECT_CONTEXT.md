# PROJECT_CONTEXT — Лесной гримуар 🌙

> Перед любыми крупными изменениями сверяйся с этим файлом.

## Цель приложения

Атмосферный личный дневник лесной ведьмы — не калькулятор и не гадалка, а уютная
магическая книга, которую хочется открыть вечером с чашкой чая. Главный акцент —
оформление, атмосфера и ощущение личной книги: тёмный лес, глубокий зелёный,
приглушённое золото, папоротники, луна, свечи, старая бумага.

Аудитория: женщины и девушки, которым близка эстетика леса, ведьмовства, уютных
ритуалов, травников, сезонности, чая и личных записей.

## Стек

- **React 18 + TypeScript + Vite** — быстрый MVP, компонентная архитектура.
- **react-router-dom (HashRouter)** — навигация; hash-роутинг устойчив к `file://`
  внутри Capacitor WebView.
- **Capacitor 8 + Android** — упор на Android-приложение. Плагины: StatusBar,
  SplashScreen. `appId: online.forestloom.grimoire`.
- **localStorage** — всё хранение данных на устройстве (первый этап). Обёртка —
  `useLocalStorage` с префиксом `grimoire:`.
- **sharp** — оффлайн-оптимизация изображений в WebP (скрипт сборки, не рантайм).

## Где что находится

| Что | Где |
|---|---|
| Экраны | `src/screens/*.tsx` |
| Маршруты | `src/App.tsx` |
| Данные (mock) | `src/data/*.ts` |
| Стили / дизайн-система | `src/styles/theme.css`, `src/styles/screens.css` |
| Хранение | `src/storage/useLocalStorage.ts`, типы — `src/storage/types.ts` |
| Изображения | `src/assets/` (готовые WebP) + `src/assets/index.ts` (реестр) |
| Нав. меню | `src/components/BottomNav.tsx` |

### Экраны
- `Home` — приветствие, дата, сезон, ближайший праздник, карта дня, быстрые тропинки.
- `Wheel` / `SabbatDetail` — Колесо года (8 праздников) + детальная карточка с
  записями по годам.
- `DailyCard` / `Archive` — карта дня (рубашка → раскрытие), лайки, история.
- `Journal` — личные записи (настроение, теги, фото, связь с праздником).
- `Wishes` — книга желаний («семена»: посажено/растёт/исполнилось/отпущено).
- `Recipes` — травник и чаи (seed-рецепты + свои).
- `Treasures` — фотозаметки/коллекция вдохновения.
- `Reminders` — авто-напоминания о празднике (7/3/0 дней) + свои; пока структура
  данных и UI (нативные уведомления подключаются позже).
- `More` — хаб второстепенных разделов.
- `Settings` — экспорт/импорт/очистка локальных данных.

### Данные
- `src/data/wheelOfYear.ts` — 8 праздников (поля `dateRange`, `lore` из WHEEL.md) +
  `nextSabbat()`, форматтеры дат, сезоны. Логотипы — `sabbatLogos` в `src/assets`.
  Пользователь может на детальной странице править своё описание и добавлять
  произвольные разделы (списки/тексты) — хранится в `grimoire:sabbatCustom`
  (тип `SabbatCustom` в `storage/types.ts`).
- `src/data/cards.ts` — карты дня + `cardForDate()` (детерминированно по дате).
- `src/data/recipes.ts` — стартовые рецепты.
- `src/data/moods.ts` — настроения для дневника.

## Как добавлять новый раздел

1. Создай `src/screens/MyScreen.tsx` (используй `PageBackground`, `PageHeader`,
   `Sheet`, `PhotoField`, классы из `screens.css`).
2. Зарегистрируй маршрут в `src/App.tsx`.
3. Добавь пункт в `src/components/BottomNav.tsx` (если основной) или в массив
   `links` в `src/screens/More.tsx` (если второстепенный).
4. Для хранения — `useLocalStorage('myKey', initial)`; тип положи в
   `src/storage/types.ts`; ключ добавь в `KEYS` в `Settings.tsx` (для бэкапа).

## Как добавлять новые карты дня

Открой `src/data/cards.ts`, добавь объект в массив `dayCards`:
```ts
{ id: 'unique', name: 'Название', text: 'Мягкий совет…',
  type: 'совет' | 'настроение' | 'задание', category: '<см. CardCategory>' }
```
Иллюстрация подбирается автоматически (`cardArtFor` по индексу, 7 артов по кругу).
Структура рассчитана на 60–100 карт; цель — расширять массив.

## Как добавлять праздники Колеса года

В `src/data/wheelOfYear.ts` добавь объект в `wheelOfYear` с полями `id, name,
date {month, day}, season, tagline, description, symbols[], colors[], plants[],
rituals[]`. Колесо, обратный отсчёт и детальная страница подхватят его сами.

## Как добавлять изображения

1. Положи исходники: фоны как `NN.jpg` в корень, карты как `N.png` в `cards/`,
   логотипы праздников как `<имя>.png` в `wheel/` (имя из `WHEEL_MAP` скрипта).
2. Запусти `npm run optimize-images` → появятся WebP в `src/assets/`.
3. Добавь импорт в `src/assets/index.ts` (`backgrounds` / `cardArt` / `sabbatLogos`).

Текущая оптимизация: фоны → 1080px WebP q72; карты → 800px q80; логотипы колеса →
480px WebP q82 с сохранением прозрачности.
Исходники (29 МБ) сжаты до ~1.9 МБ (−94%). Исходные jpg/png в сборку не входят.

## Визуальные принципы (соблюдать всегда)

- Тёмный лесной интерфейс, глубокий зелёный — основной цвет, приглушённое
  золото/латунь — акценты. Только тёмная тема.
- Шрифты: `Cormorant Garamond` (заголовки), `Spectral` (текст), `Marck Script`
  (рукописные акценты). Переменные в `theme.css`.
- Рисованные элементы, тонкие золотые рамки, мягкое свечение, старая бумага,
  папоротники, мох, луна, свечи, фонари.
- **Избегать:** кислотных цветов, белого стерильного UI, черепов/пентаграмм и
  агрессивной мистики, перегруженных эффектов, дешёвого fantasy-UI и Material.
- Тон текстов — мягкий, тёплый, женственный, чуть сказочный. Карты дня и рецепты
  не дают предсказаний/медицинских обещаний.

## Команды

```bash
npm run dev               # локальная разработка (браузер)
npm run build             # сборка в dist/
npm run optimize-images   # пересжать исходные изображения в WebP
npm run android:sync      # build + npx cap sync android
npm run android:open      # открыть проект в Android Studio
```
